Before we start modifying our gradient descent algorithm, let us first put it in a more "useful" form. Remember from the lecture on higher-order functions that we can pass any function f as parameter into another function and then just call the passed function.

def gradient_descent(grad, x0, eta, max_iter):
    
    # set starting point
    x = x0
    # do the loop max_iter times
    for _ in range(max_iter):
        # calc gradient
        gradient = grad(x)
        # if tolerance achieved stop looping
        if abs(gradient) < 1e-5: break
        # update x
        x -= eta * gradient
    
    # return x
    return x

def gradient_descent_all(grad, x0, eta, max_iter):
    
    # initialise starting x and list to store x after every step
    x = x0
    x_list = [x0]
    # loop max_iter times
    for _ in range(max_iter):
        # calc gradient
        gradient = grad(x)
        # stop looping if tolerance achieved
        if abs(gradient) < 1e-5: break
        # update x and add it to the list
        x -= eta * gradient
        x_list.append(x)
    
    # return the list
    return x_list

Nesterov-accelerated Gradient Descent¶

Finally, let's explore the effect of momentum on gradient descent.

Copy your function gradient_descent_all, and give it a new name nesterov_all. Modify the function to implement the Nesterov acceleration scheme. You now need the additional mixing parameter gamma.

Student's answer(Top)
def nesterov_all(grad, x0, eta, gamma, max_iter):
    
    # intialise x, xlist,step and previous step
    x = x0
    x_list = [x0]
    v, vprev = 0, 0
    # loop max_iter times
    for _ in range(max_iter):
        # calc predicted next x value without gradient
        xtilde = x + gamma * vprev
        # calc gradient at this predicted next x
        gradient = grad(xtilde)
        # calc the next step size, and save it to vprev
        v = gamma * v - eta * gradient
        vprev = v
        # update x and add to the list
        x += v
        x_list.append(x)
        # exit loop if tolerance is achieved
        if abs(gradient) < 1e-5: break
        # this time we only exit the loop after taking the step because
        # the gradient was an estimate for the gradient at the next x, not the current
        
    return x_list

From the very first plot where we vary both the starting points and the $\gamma$ value, we see that for starting points far away from the the minimum increasing $\gamma$ helps a lot and we get much faster convergence. This is not the case for the starting point at $x_0 = 1.3$ because here the $\gamma$s chosen are too high and thus we overshoot the minimum and ultimately take longer to converge.

In the subsequent three plots we see that varying $\eta$ while keeping $\gamma$ constant has a similar effect as in the standard non-Nesterov gradient descent algorithm. Smaller $\eta$ means we take smaller steps and convergence takes longer. We also see that if we choose $\gamma$ too big then we can overshoot the first minimum and instead land in the other minimum. In one of the plots the algorithm unluckily lands exactly on the maximum and terminates there.
